🧹 Data Cleaning & ETL OpenEnv Environment

Real‑world AI training for data preparation tasks

This environment simulates a realistic data cleaning and ETL pipeline where an AI agent must fix messy datasets using operations like filling nulls, casting columns, removing duplicates, normalizing values, merging tables, and adding derived columns. It is built for the Meta × Hugging Face OpenEnv Hackathon and fully complies with the OpenEnv specification.

🎯 Why This Environment Wins

  • Real‑world utility: Data cleaning is a daily task for data teams – not a game. Automating it saves hours of manual work.
  • Deterministic graders: Each task compares the agent's output DataFrame against a reference, producing continuous 0.0–1.0 scores with partial credit.
  • Dense reward: The agent gets immediate feedback (delta of grader score) after every action, encouraging incremental progress.
  • 3 difficulty levels: Easy (nulls & types), Medium (duplicates + normalization), Hard (multi‑table join + outliers + derived columns).
  • Fully compliant: Passes openenv validate, Docker works, baseline script reproduces scores, all required endpoints are exposed.

Observations

  • Task context: description, step count, max steps, last action feedback.
  • Dataset state: head (first 10 rows) as JSON, column dtypes, null counts, duplicate count, row count.
  • RL signals: reward from last step, done flag, current partial score (0.0–1.0).
  • Schema errors: list of detected issues (e.g., wrong dtype).

🎮 Three Tasks & Graders

Easy Null Fixer – 50‑row customer dataset with age as string, salary nulls.
Grader: 30% for correct dtypes (age int, salary int) + 70% for value match.
Baseline (GPT‑4o‑mini): 0.75 / 1.0
Max steps: 10
Medium Schema Normalizer – 200‑row orders with duplicates, inconsistent country names (USA/usa/U.S.A), wrong date format, null amounts.
Grader: 25% duplicate removal + 35% normalization + 25% dtypes + 15% null filling.
Baseline: 0.52 / 1.0
Max steps: 20
Hard ETL Pipeline – Two tables (orders + customers, 500 rows total). Join on customer_id, remove outliers (z‑score >3), add year column.
Grader: 30% join correctness + 25% outlier removal + 25% derived column presence + 20% derived column value match.
Baseline: 0.28 / 1.0
Max steps: 30

💰 Reward Shaping

  • Dense reward: reward = new_grader_score - previous_grader_score → positive for improvement, negative for regression.
  • Invalid action penalty: -0.02 (e.g., column not found, wrong parameters).
  • Terminal reward: On submit or max steps, final reward = final grader score.
  • This design gives the agent continuous feedback and prevents gaming the system.

📦 Deployment & Compliance

  • Hugging Face Space: Containerized with Docker, tagged openenv, port 7860.
  • OpenEnv spec: Implements reset(), step(), state() with typed models (DataCleanAction, DataCleanObservation).
  • Exposed endpoints: /reset, /step, /state, /baseline, /grader, /tasks, /health.
  • Baseline script: Uses OpenAI API to run a GPT‑4o‑mini agent, reproduces scores deterministically.
  • Reproducibility: All datasets are generated from a fixed seed, so the same run yields identical scores.

🚀 How to Run & Test

# 1. Clone repo and install dependencies
pip install -r requirements.txt

# 2. Run the server
uvicorn server.app:app --reload --port 7860

# 3. In another terminal, run baseline agent (needs OPENAI_API_KEY)
export OPENAI_API_KEY=your_key_here
python baseline.py

# 4. Build and run Docker
docker build -t dataclean-env ./server
docker run -p 7860:7860 dataclean-env

# 5. Test endpoints (e.g., curl http://localhost:7860/health)

The baseline.py script will print scores for all three tasks. You can also use any OpenAI‑compatible client to interact with the environment.
